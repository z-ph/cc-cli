# V1 设计文档（已废弃）

> 这是 V1 设计的归档记录。V1 使用自定义字段映射方式，已在 V2 中替换为直接使用 Claude Code 官方 settings 字段。

---

## V1 YAML Schema

```yaml
settings:
  alias: cc              # CLI 命令别名

models:
  <config-id>:           # 配置标识名，匹配 /^[a-zA-Z0-9._-]+$/
    base_url: <url>      # 必填 → 映射为 ANTHROPIC_BASE_URL
    api_key: <key>       # 必填 → 映射为 ANTHROPIC_AUTH_TOKEN
    model: <name>        # 可选 → 映射为 ANTHROPIC_MODEL
    env:                 # 可选，自定义环境变量透传
      KEY: value
```

### 字段映射关系

| YAML 字段 | 环境变量 | 必填 |
|-----------|---------|------|
| `base_url` | `ANTHROPIC_BASE_URL` | 是 |
| `api_key` | `ANTHROPIC_AUTH_TOKEN` | 是 |
| `model` | `ANTHROPIC_MODEL` | 否 |
| `env.*` | 透传到 `process.env` | 否 |

### V1 示例

```yaml
settings:
  alias: cc

models:
  glm51:
    base_url: https://open.bigmodel.cn/api/paas/v4
    api_key: sk-xxxxxxxx
    model: glm-5.1
    env:
      ANTHROPIC_BASE_URL: https://open.bigmodel.cn/api/paas/v4
      ANTHROPIC_AUTH_TOKEN: sk-xxxxxxxx

  gpt4:
    base_url: https://api.openai.com/v1
    api_key: sk-xxxxxxxx
    model: gpt-4o
    env:
      OPENAI_BASE_URL: https://api.openai.com/v1
      ANTHROPIC_AUTH_TOKEN: sk-xxxxxxxx
```

### V1 Launch 流程

1. `findConfig(configId)` 从 YAML 加载配置
2. 构建 env 对象：`{ ...process.env, ANTHROPIC_BASE_URL, ANTHROPIC_AUTH_TOKEN, ANTHROPIC_MODEL, ...config.env }`
3. `spawn('claude', [], { env, stdio: 'inherit', shell: true })`

### V1 的问题

- `base_url`/`api_key` 是自定义字段，需要手动映射到环境变量
- `env` 字段中的 `ANTHROPIC_BASE_URL` 与 `base_url` 重复，用户需要写两遍
- 无法配置 Claude Code 的其他 settings（permissions、hooks、sandbox 等）
- 没有共享默认值机制，每个 config 必须重复相同的字段

## V2 设计（当前）

```yaml
settings:
  alias: cc

base:                          # 共享默认值，深度合并到每个 config
  model: claude-sonnet-4-6
  env:
    ANTHROPIC_BASE_URL: https://api.anthropic.com
  permissions:
    allow:
      - "Bash(npm run *)"
    deny:
      - "Read(./.env)"

configs:
  <config-id>:
    model: <name>              # Claude Code 的 model 字段
    env:                       # Claude Code 的 env 字段
      ANTHROPIC_BASE_URL: <url>
      ANTHROPIC_AUTH_TOKEN: <key>
    permissions:               # Claude Code 的 permissions 字段
      allow: [...]
      deny: [...]
```

### V2 变更点

| 方面 | V1 | V2 |
|------|----|----|
| Schema 键 | `models` | `configs` |
| 共享默认值 | 无 | `base` 字段 |
| 字段名 | 自定义 (`base_url`, `api_key`) | Claude Code 官方 (`model`, `env`) |
| Launch 方式 | 设置 env vars + spawn | 写入 `settings.local.json` + spawn |
| 支持的 settings | 仅 env vars | 全部 Claude Code settings |
| 新命令 | 无 | `cc use` |
| 验证 | `base_url`/`api_key` 必填 | 无必填字段 |

### V2 Launch 流程

1. `findConfig(configId)` 从 YAML 加载配置
2. `mergeSettings(config, configId)` 深度合并 `base` + `configs[id]`
3. `writeSettingsLocal(merged)` 写入 `.claude/settings.local.json`
4. `spawn('claude', [], { stdio: 'inherit', shell: true })`

### 合并规则

- **标量**（string/number/boolean）：config 覆盖 base
- **对象**：递归合并
- **数组**：拼接并去重（与 Claude Code 自身的跨 scope 合并行为一致）

## Claude Code Settings 字段参考

> 来源：Claude Code 官方文档

### 配置文件位置

| Scope | 路径 | 共享 |
|-------|------|------|
| Managed | `managed-settings.json` / plist / registry | 组织级 |
| User | `~/.claude/settings.json` | 否 |
| Project | `.claude/settings.json` | 是（git） |
| Local | `.claude/settings.local.json` | 否（gitignored） |

### 优先级（高→低）

1. Managed settings
2. 命令行参数
3. Local (`.claude/settings.local.json`)
4. Project (`.claude/settings.json`)
5. User (`~/.claude/settings.json`)

### 常用 settings 字段

| Key | 说明 | 示例 |
|-----|------|------|
| `model` | 使用的模型 | `"claude-sonnet-4-6"` |
| `env` | 环境变量 | `{"FOO": "bar"}` |
| `permissions.allow` | 允许的权限规则 | `["Bash(npm run *)"]` |
| `permissions.deny` | 拒绝的权限规则 | `["Read(./.env)"]` |
| `permissions.defaultMode` | 默认权限模式 | `"acceptEdits"` |
| `hooks` | 生命周期钩子 | 见 hooks 文档 |
| `sandbox.enabled` | 启用沙箱 | `true` |
| `sandbox.filesystem.allowWrite` | 沙箱可写路径 | `["/tmp/build"]` |
| `sandbox.network.allowedDomains` | 沙箱允许域名 | `["github.com"]` |
| `apiKeyHelper` | API Key 生成脚本 | `/bin/generate_key.sh` |
| `alwaysThinkingEnabled` | 启用扩展思考 | `true` |
| `effortLevel` | 努力程度 | `"high"` |
| `language` | 响应语言 | `"japanese"` |
| `attribution` | git 提交归属 | `{"commit": "...", "pr": ""}` |

### 权限规则语法

格式：`Tool` 或 `Tool(specifier)`

| 规则 | 效果 |
|------|------|
| `Bash` | 匹配所有 Bash 命令 |
| `Bash(npm run *)` | 匹配 `npm run` 开头的命令 |
| `Read(./.env)` | 匹配读取 `.env` 文件 |
| `WebFetch(domain:example.com)` | 匹配 fetch 到 example.com |

### 相关环境变量

| 变量 | 说明 |
|------|------|
| `ANTHROPIC_BASE_URL` | API 基础地址 |
| `ANTHROPIC_AUTH_TOKEN` | API 认证 token |
| `ANTHROPIC_MODEL` | 模型名称 |
| `ANTHROPIC_API_KEY` | API key（注意：本工具使用 AUTH_TOKEN） |
| `CLAUDE_CODE_USE_BEDROCK` | 使用 AWS Bedrock |
| `CLAUDE_CODE_USE_VERTEX` | 使用 Google Vertex |
| `CLAUDE_CODE_DEBUG` | 调试模式 |

## 项目架构

```
bin/cc.js              # CLI 入口，Commander.js
src/commands/
  launch.js            # cc <id> — 合并配置 + 写 settings + 启动 claude
  use.js               # cc use <id> — 合并配置 + 写 settings（不启动）
  add.js               # cc add <id> — 交互式添加配置
  edit.js              # cc edit <id> — 交互式编辑配置
  list.js              # cc list — 列出配置
  remove.js            # cc remove <id> — 删除配置
  alias.js             # cc alias [name] — 查看/修改别名
src/config/
  loader.js            # YAML 读写、findConfig（三级查找：custom > local > global）
  validator.js         # validateConfigId + validateConfigEntry
  merger.js            # deepMerge + mergeSettings + writeSettingsLocal
  env-registry.js      # 环境变量注册表（内置 22 个 Claude Code 变量 + 用户自定义）
tests/                 # Jest 测试（add/edit 交互式命令排除在 coverage 外）
```

### 关键设计决策

- `api_key` 映射到 `ANTHROPIC_AUTH_TOKEN`（不是 `ANTHROPIC_API_KEY`）— V1 的 post-launch fix
- `add` 默认保存到 local config（`./.claude/models.yaml`），其他命令默认 global
- `add.js` 和 `edit.js` 排除在 Jest coverage 之外（inquirer 交互式）
- env-registry 支持全局/本地两个 YAML 文件，与主配置文件独立
- 数组合并采用 concat + dedup，与 Claude Code 跨 scope 合并行为一致

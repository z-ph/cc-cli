## YAML Schema

```yaml
settings:
  alias: cc                # CLI 命令别名（非 Claude Code 字段）

base:                      # 共享默认值，深度合并到每个 config
  model: claude-sonnet-4-6
  env:
    ANTHROPIC_BASE_URL: https://api.anthropic.com
  permissions:
    allow:
      - "Bash(npm run *)"
    deny:
      - "Read(./.env)"

configs:
  <config-id>:             # 配置标识名，匹配 /^[a-zA-Z0-9._-]+$/
    model: <name>          # Claude Code 的 model 字段
    env:                   # Claude Code 的 env 字段
      ANTHROPIC_BASE_URL: <url>
      ANTHROPIC_AUTH_TOKEN: <key>
    permissions:           # Claude Code 的 permissions 字段
      allow: [...]
      deny: [...]
    # 以及其他 Claude Code settings 字段（hooks, sandbox 等）
```

直接使用 Claude Code 官方 settings 字段，无自定义映射。每个 config 本质上是一个 Claude Code settings 对象。

### 合并规则

启动或 `use` 时，`base` 和对应 config 深度合并：

- **标量**（string/number/boolean）：config 覆盖 base
- **对象**：递归合并
- **数组**（permissions.allow 等）：拼接并去重

### 示例

```yaml
settings:
  alias: cc

base:
  permissions:
    allow:
      - "Bash(npm run *)"
    deny:
      - "Read(./.env)"

configs:
  glm51:
    model: glm-5.1
    env:
      ANTHROPIC_BASE_URL: https://open.bigmodel.cn/api/paas/v4
      ANTHROPIC_AUTH_TOKEN: sk-xxxxxxxx

  gpt4o:
    model: gpt-4o
    env:
      ANTHROPIC_BASE_URL: https://api.openai.com/v1
      ANTHROPIC_AUTH_TOKEN: sk-xxxxxxxx
    permissions:
      allow:
        - "Bash(*)"

  local:
    model: llama3.1
    env:
      ANTHROPIC_BASE_URL: http://localhost:11434/v1
      ANTHROPIC_AUTH_TOKEN: ollama
```

## 命令

| 命令 | 说明 |
|------|------|
| `cc <config-id>` | 合并配置 → 写入 `.claude/settings.local.json` → 启动 Claude Code |
| `cc use <config-id>` | 合并配置 → 写入 settings 文件（不启动） |
| `cc use <config-id> -g` | 写入 `~/.claude/settings.json` |
| `cc list` | 列出所有配置 |
| `cc add <config-id>` | 交互式添加配置 |
| `cc edit <config-id>` | 交互式编辑配置 |
| `cc remove <config-id>` | 删除配置 |
| `cc alias [name]` | 查看/修改命令别名 |

全局选项：`-t, --target <file>` 指定自定义 YAML 配置文件。`-g, --global` 操作全局配置。

## 配置文件查找

优先级（高→低）：

1. `-t` 指定的自定义路径
2. `./.claude/models.yaml`（当前目录）
3. `~/.claude/models.yaml`（用户目录）

`findConfig()` 按优先级查找，只选择包含目标 configId 的文件。

---

## Launch 流程

```
cc <config-id>
  │
  ├─ findConfig(configId)         → 定位 YAML 文件，加载 config
  ├─ mergeSettings(config, id)    → 深度合并 base + configs[id]
  ├─ writeSettingsLocal(merged)   → 写入 ./.claude/settings.local.json
  └─ spawn('claude')              → Claude Code 读取 settings 文件启动
```

## Claude Code Settings 字段参考

> 来源：Claude Code 官方文档

### settings 文件位置

| Scope | 路径 | 说明 |
|-------|------|------|
| Managed | `managed-settings.json` / plist / registry | 组织级，不可覆盖 |
| User | `~/.claude/settings.json` | 全局个人 |
| Project | `.claude/settings.json` | 项目共享（git） |
| Local | `.claude/settings.local.json` | 项目个人（gitignored） |

### 优先级（高→低）

1. Managed settings
2. 命令行参数
3. Local (`.claude/settings.local.json`)
4. Project (`.claude/settings.json`)
5. User (`~/.claude/settings.json`)

### 常用字段

| Key | 说明 | 示例 |
|-----|------|------|
| `model` | 使用的模型 | `"claude-sonnet-4-6"` |
| `env` | 环境变量 | `{"FOO": "bar"}` |
| `permissions.allow` | 允许规则 | `["Bash(npm run *)"]` |
| `permissions.deny` | 拒绝规则 | `["Read(./.env)"]` |
| `permissions.defaultMode` | 默认权限模式 | `"acceptEdits"` |
| `hooks` | 生命周期钩子 | 见 hooks 文档 |
| `sandbox.enabled` | 启用沙箱 | `true` |
| `sandbox.filesystem.allowWrite` | 沙箱可写路径 | `["/tmp/build"]` |
| `sandbox.network.allowedDomains` | 沙箱允许域名 | `["github.com"]` |
| `apiKeyHelper` | API Key 生成脚本 | `/bin/generate_key.sh` |
| `alwaysThinkingEnabled` | 启用扩展思考 | `true` |
| `effortLevel` | 努力程度 | `"high"` |
| `language` | 响应语言 | `"japanese"` |
| `attribution` | git 归属 | `{"commit": "...", "pr": ""}` |

### 权限规则语法

格式：`Tool` 或 `Tool(specifier)`

| 规则 | 效果 |
|------|------|
| `Bash` | 所有 Bash 命令 |
| `Bash(npm run *)` | `npm run` 开头的命令 |
| `Read(./.env)` | 读取 `.env` 文件 |
| `WebFetch(domain:example.com)` | fetch 到 example.com |

### 相关环境变量

| 变量 | 说明 |
|------|------|
| `ANTHROPIC_BASE_URL` | API 基础地址 |
| `ANTHROPIC_AUTH_TOKEN` | API 认证 token |
| `ANTHROPIC_MODEL` | 模型名称 |
| `CLAUDE_CODE_USE_BEDROCK` | 使用 AWS Bedrock |
| `CLAUDE_CODE_USE_VERTEX` | 使用 Google Vertex |
| `CLAUDE_CODE_DEBUG` | 调试模式 |

## 设计决策

- 字段直接使用 Claude Code 官方名称，无自定义映射
- `add` 默认保存到 local config，其他命令默认 global
- `add.js` / `edit.js` 排除在 Jest coverage 之外（inquirer 交互式）
- 数组合并采用 concat + dedup，与 Claude Code 跨 scope 合并行为一致
- `use` 命令支持 `-g` 写入 `~/.claude/settings.json`，否则写入项目级 `settings.local.json`
- 写入前自动备份已有 settings 文件为 `.bak`
